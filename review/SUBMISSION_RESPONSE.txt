## 3. Rate Limiting Implementation ✅

**Dual-Layer Protection:**

### A. Application Layer (Express.js)
File: src/Infrastructures/http/middlewares/rateLimiter.js

- In-memory sliding window algorithm
- 90 requests/minute (production)
- 1000 requests/minute (development)
- Applied to /threads endpoints
- HTTP 429 response with JSON error message

**Integration:**
- src/Infrastructures/http/createServer.js
- Environment-aware configuration

### B. Web Server Layer (Nginx)
File: nginx.conf

```
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=90r/m;
limit_req zone=api_limit burst=10 nodelay;
```

**Configuration:**
- 90 requests per minute per IP
- 10MB zone (160,000 IP capacity)
- Burst handling: 10 excess requests
- Scope: /threads, /api endpoints

**DDoS Protection:** ✅ ACTIVE

**Documentation:**
- review/Menerapkan Limit Access pada Server Nginx.txt
- docs/RATE_LIMITING.md
- nginx.conf (documented configuration)
